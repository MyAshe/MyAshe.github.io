Android drawable state 属性简介

在定义一个 drawable 的时候可以通过在 xml 里面定义的 drawable 对象，使得一个对象能在不同的状态下显示不同的内容，例如 Button，它有 pressed，focused，等状态，通过selector 选择，可以为每种状态提供不同的内容。

这些属性的意义：
android:drawable 放一个drawable里面的资源。
android:state_pressed 是否按下，如一个按钮触摸或者点击。
android:state_focused 是否取得焦点，比如用户选择了一个文本框。
android:state_hovered 光标是否悬停，通常与focused state相同。
android:state_selected 被选中，它与focus state并不完全一样，可能选中整个ListView。
android:state_checkable 组件是否能被check。如：RadioButton是可以被check的。
android:state_checked 被打钩了，如：一个RadioButton可以被打钩了。
android:state_enabled 能够接受触摸或者点击事件。
android:state_activated 被激活(例子)。
android:state_window_focused 应用程序是否在前台。

这些可以用在ListView中，如此配置：
1：在listview中配置 android:listSelector = “@drawable/example”
2：在listview的item中添加属性 android:background = “@drawable/example”
3：在java代码中使用 Drawable drawable = getResources().getDrawable(R.drawable.example); (列表有时候会变黑，需要加上：android:cacheColorHint=”@android:color/transparent”)
